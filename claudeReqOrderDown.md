# [교정신청서] 작업 요청서

## 1. 작업 개요
교정신청서 엑셀 다운로드 기능을 구현해 주세요.

기본적으로 제공된 양식을 기준으로 데이터를 삽입한 뒤, 다운로드 가능한 형태로 처리하는 작업입니다.

---

## 2. 참고 양식
- 기준 양식 파일:
  - `cali\backend\src\main\resources\static\document\order.xlsx`

위 파일은 참고용이며, 먼저 탐색해줄 것. 실제 작업은 스토리지 /dev/env/order.xlsx 를 임시폴더에내려받아서 진행함.

---

## 3. 구현 방식

### 3-1. 엑셀 처리 라이브러리 주입
- 엑셀에 데이터를 삽입할 수 있도록 적절한 라이브러리를 프로젝트에 주입해 주세요.
- 현재 프로젝트 구조와 호환되는 안정적인 방식으로 선택해 주세요.

### 3-2. 파일 처리 흐름
엑셀 다운로드는 아래 방식으로 구현해 주세요.

1. 스토리지에 있는 교정신청서 양식을 로컬의 `temp` 디렉토리 아래로 내려받기
2. 이때 작업용 디렉토리는 **UUID 형태의 디렉토리명**으로 생성할 것
3. 해당 UUID 디렉토리 안에 교정신청서 엑셀 파일을 저장할 것
4. 내려받은 엑셀 파일을 열어서 데이터 삽입
5. 가공된 엑셀 파일을 사용자에게 다운로드 응답
6. 다운로드 호출이 끝나면, 작업에 사용한
   - UUID 디렉토리
   - 임시 다운로드한 엑셀 파일
   만 삭제할 것

### 3-3. 삭제 처리 주의사항
- 임시 파일 정리 시 **작업 대상 디렉토리와 파일만 삭제**해야 합니다.
- 절대로 상위 디렉토리나 다른 작업 파일까지 삭제되지 않도록 안전하게 구현해 주세요.

---

## 4. 데이터 삽입 규칙

### 4-1. 상단 정보
- 상단 정보는 **해당 접수 데이터에서 삽입 가능한 값만 넣을 것**
- 현재 기준으로 알 수 없는 값은 **억지로 넣지 말고 비워둘 것**
- `신청일` 은 **현재 시점 기준**으로 아래 형식으로 삽입할 것
  - `YYYY. MM. DD`

예시:
- `2026. 03. 10`

### 4-2. 수정 금지 영역
- 양식 내 **중간의 `적합성 진술` 부터 `검토사항` 까지의 영역은 건드리지 말 것**
- 즉, 해당 구간의 문구/병합/서식/행 구조는 변경하지 않도록 주의할 것

---

## 5. 교정신청 리스트 삽입 규칙

### 5-1. 기본 삽입 수량
교정신청 리스트는 기본적으로 아래 수량을 수용합니다.

- 1페이지: 10개
- 2페이지: 22개

즉, 기본 양식 기준으로 **총 32개의 성적서 데이터**를 삽입할 수 있습니다.

### 5-2. 대상 데이터 기준
- 해당 접수의 **성적서 데이터**를 삽입할 것
- 정렬 기준은 **성적서번호 순서**
- **재발행 성적서는 제외**
- **대행은 포함**
- 따라서 `report_type` 으로 대행 여부를 제외하는 로직은 넣지 말 것

### 5-3. 기존 함수 활용
- 기존에 해당 접수 건의 성적서 데이터를 가져오는 함수가 있다면 **그 함수를 우선 사용할 것**
- 단, **기존 함수의 수정은 하지 말 것**
- 이유:
  - 해당 함수를 다른 로직에서도 사용할 수 있으므로
  - 수정으로 인해 다른 기능에 오류가 발생하면 안 됨

즉, 가능하면 기존 조회 함수를 **재사용**하고,
부득이하게 추가 가공이 필요하면 **별도 로직에서 처리**해 주세요.

---

## 6. 32건 초과 시 처리 방식

### 6-1. 행 추가 필요
- 성적서 개수가 **32개를 초과하면**, 데이터를 삽입하기 전에 먼저 **행 추가**를 진행할 것
- 행 추가 위치는 **교정신청 리스트의 마지막 행 쪽**
- 엑셀 조작 시, 단순 문자열 치환이 아니라 **행 추가 함수 기반**으로 처리해 주세요

### 6-2. 처리 순서
반드시 아래 순서로 진행해 주세요.

1. 성적서 데이터 개수 확인
2. 32개 초과 여부 확인
3. 초과 시, 리스트 마지막 행 부근에서 필요한 만큼 행 추가
4. 그 뒤에 성적서 데이터 삽입

### 6-3. 주의사항
- 행 추가 시 기존 양식의
  - 테두리
  - 셀 스타일
  - 높이
  - 병합 상태
  - 수식/인쇄 레이아웃 영향
  등을 최대한 깨지 않도록 처리해 주세요.
- 특히 출력용 양식이므로 **서식 유지**를 신경 써 주세요.

---

## 7. 이미지 삽입
- 양식의 **좌측 상단에는 회사 로고를 삽입**해야 합니다.
- 로고는 양식 레이아웃을 해치지 않도록 **적절한 크기**로 맞춰서 삽입해 주세요.
- 원본 비율이 크게 깨지지 않도록 처리해 주세요.

---

## 8. 로그 처리
- 엑셀 다운로드 작업에 대한 **로그를 남길 것**
- 최소한 아래 정보는 로그에 남길 수 있도록 고려해 주세요.
  - 어떤 접수 건에 대해 다운로드했는지
  - 누가 다운로드했는지
  - 언제 다운로드했는지
  - 어떤 기능(교정신청서 다운로드)인지

기존 프로젝트의 로그 방식이 있다면 그 규칙에 맞춰 주세요.

---

## 9. 서비스 구조 제안
- 엑셀 데이터 삽입과 관련해서는 별도의 `ExcelServiceImpl` 선언도 고려해 주세요.
- 즉, 컨트롤러 / 서비스 / 엑셀처리 책임이 한 곳에 과도하게 몰리지 않도록 구조를 검토해 주세요.

예를 들면 아래와 같은 방향을 고려할 수 있습니다.

- 접수/성적서 조회 담당 서비스
- 엑셀 템플릿 가공 담당 서비스 (`ExcelServiceImpl` 등)
- 파일 다운로드 응답 및 temp 정리 담당 로직

구조 분리가 더 적절하다면 그 방향으로 제안해 주세요.

---

## 10. 구현 시 유의사항

### 10-1. 데이터 우선 원칙
- 상단 정보는 접수 데이터 기준으로 채울 수 있는 것만 채울 것
- 알 수 없는 값은 비워둘 것
- 임의값/하드코딩 금지

### 10-2. 기존 로직 영향 최소화
- 기존 성적서 조회 함수를 사용할 수 있으면 사용할 것
- 기존 함수 수정 금지
- 다른 로직에 영향을 주지 않는 방향으로 구현할 것

### 10-3. temp 파일 정리
- 다운로드 이후 temp 정리 로직 필수
- 단, 현재 작업에서 생성한 UUID 디렉토리 범위 내에서만 삭제할 것

### 10-4. 예외 처리
- 다운로드 중 오류 발생 시 예외처리를 명확히 할 것
- temp 파일/디렉토리가 남지 않도록 정리 로직도 함께 고려할 것
- 단, 정리 과정에서도 다른 디렉토리를 침범하지 않도록 주의할 것

---

## 11. 작업 순서 제안

### 1단계. 라이브러리 및 구조 준비
- 엑셀 처리 라이브러리 추가
- 필요 시 `ExcelServiceImpl` 구조 설계

### 2단계. 템플릿 다운로드 및 temp 작업 구조 구현
- 스토리지 → temp/UUID 디렉토리 다운로드
- 작업 파일 로딩 구조 구현
- 다운로드 후 정리 로직 구현

### 3단계. 상단 정보 삽입
- 접수 데이터 기반 상단 정보 매핑
- `신청일` 현재일자 포맷 적용

### 4단계. 교정신청 리스트 삽입
- 성적서 데이터 조회
- 성적서번호 순 정렬
- 재발행 제외
- 32건 이하/초과 분기 처리
- 초과 시 행 추가 후 데이터 삽입

### 5단계. 회사 로고 삽입
- 좌측 상단 이미지 삽입
- 크기 및 배치 보정

### 6단계. 다운로드 및 로그 처리
- 최종 파일 응답
- 다운로드 로그 기록
- temp 정리

## 12. 클로드가 먼저 확인하거나 제안해줄 사항
작업 전에 아래 항목 중 불명확한 점이 있으면 먼저 질문해 주세요.

- 스토리지에 있는 교정신청서 원본 경로
- 회사 로고 이미지의 저장 위치 및 조회 방식
- 성적서 데이터에서 양식 각 컬럼에 정확히 어떤 값을 매핑해야 하는지
- 재발행 성적서 제외 기준 컬럼/조건
- 성적서번호 정렬 기준 필드명
- 다운로드 로그 저장 방식
- temp 디렉토리 기준 경로
- 행 추가 시 복제해야 하는 기준 행 번호 또는 기준 구간
- 다운로드 후 파일 삭제 시점
  - 응답 직후인지
  - 스트림 종료 이후인지

또한 더 나은 방향이 있다면 아래 형식으로 제안해 주세요.

- 현재 요구사항 기준 구현안
- 더 나은 대안
- 그 이유
- 적용 시 영향 범위

---

## 13. 최종 요청
위 요구사항을 모두 반영하여 교정신청서 엑셀 다운로드 기능을 구현해 주세요.

특히 아래 사항은 반드시 지켜 주세요.

- `order.xlsx` 양식 기준으로 작업
- 엑셀 데이터 삽입 라이브러리 주입
- 스토리지 → temp/UUID 디렉토리 다운로드 방식 사용
- 작업 완료 후 작업 대상 temp 파일/디렉토리만 삭제
- 상단 정보는 접수 데이터 기준으로만 채우고, 모르는 값은 비워두기
- `신청일` 은 현재일자 `YYYY. MM. DD` 형식 사용
- `적합성 진술 ~ 검토사항` 구간은 수정 금지
- 성적서 데이터는 성적서번호 순으로 삽입
- 재발행 성적서는 제외
- 대행은 포함
- 기존 성적서 조회 함수가 있으면 재사용, 수정은 금지
- 회사 로고는 좌측 상단에 적절한 크기로 삽입
- 32건 초과 시 행 추가 후 데이터 삽입
- 다운로드 로그 기록
- 필요 시 `ExcelServiceImpl` 별도 선언 검토
