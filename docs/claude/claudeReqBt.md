# [출장일정] 작업 요청서

## 1. 작업 배경
- 교정업무의 경우, 본사에서 교정을 하는 경우(교정접수 유형의 '고정표준실'에 해당)도 있지만 대개 신청업체에 출장을 나가서 교정을 진행하는 경우가 빈번(접수유형의 '현장교정'으로 분류)
- 그래서 calendar api를 활용하여 출장일정을 등록/수정을 포함하여 관리할 수 있는 메뉴가 필요. 
- 현재는 menu테이블에 /cali/businessTrip 을 URL로 등록되어 있고, menu_code 의 값은 business_trip 입니다. '교정관리'의 하위 메뉴에 속하며 2depth 중 첫 번째 입니다.
- menu테이블에만 존재하기 때문에 관련 html, js, css, 비즈니스 로직  또한 전무한 상태이며, DB 스키마도 없습니다. 

---

## 2. 작업 원칙
- 본 요청서를 먼저 확인한 뒤, **확인받아야 할 사항이 있다면 반드시 피드백을 요청할 것**
- **작업 단위를 분리하여 단계적으로 진행할 것**
- 우선은 **조회 로직과 화면구성까지만 구현**할 것
- 저장 버튼을 눌렀을 때는 실제 저장이 아니라 **`구현 준비중입니다` 토스트(gToast)** 를 띄우도록 할 것

### 3.2 화면 UI 구성
- 출장일정 메인 화면은 full calendar api를 활용하여 화면에 렌더링
  - 관련 라이브러리 의존성 추가 필요
  - 일자별로 일단 기본적으로 최대 7개까지 표시 & 넘어가면 '더보기' 활성화
  - 캘린더의 일자를 더블클릭하면 gModal을 통해서 출장일정을 등록/수정 할 수 있는 모달을 띄운다
  - 캘린더 우측 상단에 '일정등록' 버튼을 클릭하면 출장일정을 등록하는 gModal을 호출
    - 마찬가지로 캘린더에 표시된 일정을 클릭하면 출장일정을 수정하는 모달을 호출한다.
    - 캘린더에서 일자별 빈 영역을 더블클릭?하면 등록하는 모달 호출
  - 일자가 등록이 되면,  '유형'이 맨 위에 표시되고(일단은 임의로 '일정'으로 표시 추후 type에 대한 정의가 생기면 수정 예정)
  - 그다음 행에 일정제목(title)이 표시되고, 그 다음 행엔 출장하는 사람들을 표시한다.

- 출장일정 등록/수정 모달
  - gmodal은 가장 큰 사이즈로 호출  등록 시 '출장일정 등록', 수정 시 모달의 타이틀은 '출장일정 수정'
  - 좌측엔 그리드 2개로 구성되어 있고, 우측엔 출장에 대한 정보가 기록된다.
    - 그리드 2개는 '표준장비 리스트'(위), '출장차량'(아래)로 구성
    - 표준장비 리스트는 성적서 수정(reportModify)와 동일한 형태이며, 출장일정 등록 시, standard_equipment_ref 테이블에 ref_table 값으로 business_trip로 저장된다. ref_table_id는 출장일정의 id가 된다.
    - 출장차량 리스트의 경우 현재 car가 없기 때문에 컬럼만 표시
      - 출장차량 컬럼: 차명, 차량번호, 년식, 
    - 표준장비 리스트
      - 관리번호, 기기명, 제작회사, 형식, 기기번호, 차기교정일, 교정기관 표시
      - 표준장비 리스트 그리드의 경우, 성적서수정에서의 그리드와 마찬가지로 드래그로 행의 순서를 변경할 수 있음. 그리드 순서대로 equipment_ref 테이블의 seq에도 기록된다.
    - 두 그리드 각각 우측 상단에 '조회'버튼 활성화 (차량정보는 클릭시 gToast를 통해 '구현 준비중입니다'
    - 표준장비의 경우엔 성적서수정 모달에서 표준장비를 조회할 때와 동일하게 모달을 호출하여 선택한다.
  - 우측 출장일정 정보 (아래 작성한 순서대로 위에서부터 구성된다고 보면 됨. 1개면 1개가 한 행을 다 차지)
    - 일정제목(title) / 일정유형(select. 아직 유형에 대한 명세가 없기 때문에 '없음'으로 표시)
